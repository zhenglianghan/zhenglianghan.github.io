阿里云服务centos系统下部署tomcat项目教程

2018-10-30   服务器 

1 购买服务器

https://www.aliyun.com/product/ecs?spm=5176.8142029.selected.1.e9396d3eHuaYMw

学生优惠服务器：

https://promotion.aliyun.com/ntms/act/campus2018.html

选操作系统时选centos

2开放端口

阿里云服务器默认只打开了22端口用作远程连接，必须要在控制台里设置才能打开其它端口（服务器中设置无用）：

https://jingyan.baidu.com/article/03b2f78c31bdea5ea237ae88.html

这里我们需打开8080端口（tomcat使用）；3306端口(mysql使用)；20,21端口（scp使用，拿来传文件的）

3连接到服务器

虽然可以在阿里云的控制台远程连接到服务器，但是在那里的复制粘贴操作很麻烦，而且不能输入和显示中文，所以建议安装SSH后在本地连接到服务器。

注意，这里是本地安装ssh，用来连接到远程服务器，而不是在服务器上安装ssh

windows:http://www.freesshd.com/index.php?ctt=download

ubuntu:https://jingyan.baidu.com/article/9c69d48fb9fd7b13c8024e6b.html

之后在命令行输入:

ssh root@120.79.44.***

这里后面的IP为服务器的公网IP，可以在控制台查看到。

完了之后输入root账号的密码，就连接到了远程服务器了。

4安装java

sudo yum install java-1.8.0-openjdk
sudo yum install java-1.8.0-openjdk-devel

5安装启动tomcat

下载tomcat

wget http://mirror.bit.edu.cn/apache/tomcat/tomcat-8/v8.5.28/bin/apache-tomcat-8.5.28.tar.gz

要安装其它版本看这个教程：

https://blog.csdn.net/qq_29291085/article/details/77389361

下载完成后解压：

tar -zxvf apache-tomcat-8.5.28.tar.gz

注意这里的文件名，不同版本号下下来的文件名可不同

切换到tomcat的bin目录下：

cd /root/apache-tomcat-8.5.28/bin

之后输入ls，可以看到有一个startup.sh文件，运行它来启动tomcat:

./startup.sh

之后访问主机地址(公网IP):8080 即可访问到tomcat页面了。

6安装启动数据库

这里安装mariadb，是mysql的一个子版本

sudo yum install mariadb-server

启动数据库服务

service mariadb start

开机自启动

systemctl enable mariadb

设置mysql用户密码

mysqladmin -u root password 'mypassword'

7传输war包与sql文件到服务器

eclispe导出war包：

https://jingyan.baidu.com/article/624e74597ccebe34e9ba5a58.html

idea导出war包：

https://jingyan.baidu.com/article/a24b33cd10adf719fe002ba1.html

导出sql文件：

https://blog.csdn.net/kwame211/article/details/75635463

之后将这两个文件使用scp命令，或者下载winscp（windows向linux传输文件神器），传输到远程服务器上。

先在远程服务器上运行一下sql文件(假设文件名为test.sql,放在了root目录下):

mysql -u root -p
create database test;
use test;
source /root/test.sql;

结束后查一下表什么的看有没有出什么错。如果有中文乱码问题，百度一哈，或者去安装mysql而不用mariadb，一般来说是没啥问题的。

将war包，假设文件名为test.war，放到tomcat下的webapp目录下。

之后切换到tomcat目录的bin目录下

cd /root/apache-tomcat-8.5.28/bin

重启tomcat：

./shutdown.sh
./startup.sh

8访问网站

访问公网IP:8080/war包名，就能看到部署上去的网站了，通常从开发环境到生产环境会出些小问题，耐心解决一下即可。

9实时查看日志

切换到logs目录：cd tomcat/logs

然后：tail -f catalina.out
这样运行时就可以实时查看运行日志了，看是否运行报错
