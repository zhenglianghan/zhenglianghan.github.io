DOM中获取元素的实际大小

2018-10-06   JavaScript CSS 

此篇博客讲解如何使用JS获取元素的实际内容大小，即元素不包括内外边距和边框的大小。如下图中设置了内边距和边框的img元素中实际花朵图片的大小。

为什么不用Element.style.width/heigh?

Element.style中保存的是元素的内联样式，也就是写在标签的style特性中的样式，而影响到元素的样式通常是内联样式、嵌入样式、外部样式层叠而来的。在没有给标签设定内联样式的宽高时,Element.style.width和Element.style.height返回空字符串。

为什么不用offsetWidth/Heigh,clientWidth/Heigh,scrollWidth/Heigh,getBoundingClientRect()？
  1. offsetHeight/Width用于描述元素外尺寸，是指元素内容+内边距+边框，不包括外边距和滚动条部分
  2. clientHeight/Width用于描述元素内尺寸，是指元素内容+内边距大小，不包括边框（IE下实际包括）、外边距、滚动条部分
  3. scrollHeight/Width : 在没有滚动条的情况下，元素内容+内边距的部分
  4. getBoundingClientRect()方法。这个方法会返回一个矩形对象，包括4个属性：left,top,right,和bottom（现在的浏览器还会有height和width）。这些属性给出了元素（包含内边距和边框）在页面中相对于视口的位置

示例：

可以看到，这些属性或方法得到的值，要么包括了内边距，要么包括了内边距加边框，无法获取到元素实际内容的大小。

使用计算样式获取标准盒子的实际大小

JS中使用document.defaultView.getComputedStyle()方法可以获得指定元素的计算样式（所有样式表层叠而来并影响到当前元素的样式），这个方法返回一个CSSStyleDeclaration对象，其中包含当前元素的所有计算的样式。IE不支持这个方法，但是有个currentStyle属性。记住所有计算得样式都是只读的。使用示例：

HTML:

<!DOCTYPE html>
<html>
<head>
<style>
    div#div1{
        background: #FFB6C1;
        padding : 30px;
        border : 20px solid #FF0000;
        margin : 50px;
        height : 200px;
        width : 300px;
    }
</style>
</head>
<body>
    <div id="div1"></div>
</body>
</html>

此盒子在浏览器中的表现：

JS :

var div1 = document.getElementById("div1");
var style1 = document.defaultView.getComputedStyle(div1)||div1.currentStyle
alert(style1.width);  //"300px"
alert(style1.heigh);  //"200px"

可以看到此处正确的获取到了这个标准盒子的实际内容宽度与高度。当然使用计算样式还可以获取到元素的其它样式。

使用计算样式获取IE盒子的实际大小

HTML:

<!DOCTYPE html>
<html>
<head>
<style>
    div#div2{
        box-sizing: border-box;
        background: #FFB6C1;
        padding : 30px;
        border : 20px solid #FF0000;
        margin : 50px;
        height : 200px;
        width : 300px;
    }
</style>
</head>
<body>
    <div id="div2"></div>
</body>
</html>

这里设置了盒子的box-sizing样式值为border-box,使此盒子成为IE盒子。此盒子在浏览器中的表现：

JS:

var div2 = document.getElementById("div2");
var style2 = document.defaultView.getComputedStyle(div2)||div2.currentStyle
alert(style2.width);  //"300px"
alert(style2.heigh);  //"200px"

可以看到，此盒子的内容宽度实际为200px，而得到的计算宽度为300px，这是因为在IE盒子模型中，

width = content_width + padding + border。这个width即为计算样式的width，也就是在样式表中设定的width。而在标准盒子模型中,width = content_width。

于是此处我们应该这样得到盒子的内容宽高：

var contentWidth = parseFloat(style2.width)-parseFloat(style2.borderLeftWidth)-parseFloat(style2.borderRightWidth)+"px";
var contentHeigh = parseFloat(style2.heigh)-parseFloat(style2.borderTopWidth)-parseFloat(style2.borderBottomWidth)+"px";

那么能不能用一个函数搞定这两种情况呢？先判断盒子是什么模型，然后再进行相应的计算。可惜判断盒子是什么模型是十分困难的：

ie8以上都是w3c标准盒模型 ie5极其以下都是ie盒子模型，ie6、ie7、ie8在混杂模式下ie盒模型，在标准模式下是w3c标准盒模型，(注意：ie6在混杂模式下一定是Ie盒模型，而ie7、ie8在混杂模式下不一定是ie盒模型)

另外我们还要考虑到CSS3中的box-sizing属性可以自由的切换盒子的模型。且老版本的IE不支持CSS3，也就没办法单靠这个属性的值来判断盒子的模型，同时这个属性还能从父元素中继承……
